# 00. JS/TS 速览与语言对比（面向 C#/C++）

本章目标：你不需要成为前端工程师，也能看懂和修改 OpenClaw 主干代码。

## 1) JavaScript 是什么，能做什么

- JavaScript 是动态语言，运行在 V8/Node.js（也可在浏览器）
- 在本项目中，JS 主要用于：CLI、网关服务、消息处理、工具调用编排
- 擅长：I/O 密集、网络服务、脚本化自动化、生态集成
- 不擅长：极限 CPU 计算、强实时、高性能数值计算（这类更适合 C++/Rust）

## 2) TypeScript 是什么，为什么项目用它

- TypeScript = JavaScript + 静态类型（编译时）
- 运行时仍是 JS（Node 执行编译结果）
- 在大型工程中的价值：
  - 降低重构风险
  - 提高 API 边界可读性
  - 让 IDE 导航/补全接近 C# 体验

## 3) 和 C#/C++ 的核心对比

| 维度 | C# | C++ | JS/TS (本项目) |
| --- | --- | --- | --- |
| 执行模型 | CLR + 线程池 | 原生线程/内存控制 | Node 事件循环 + async/await |
| 类型系统 | 强类型(运行时也有类型信息) | 强类型(编译期) | TS 强类型(编译期)，运行时弱类型 |
| 内存 | GC | 手工/RAII/智能指针 | GC |
| 并发 | Task/async + 线程 | 线程/协程 | 单线程事件循环 + 非阻塞 I/O |
| 适合场景 | 企业应用、后端 | 系统/高性能 | 网关/脚本/编排/工具生态 |

## 4) 你在本项目要重点适应的语法与模式

- `async/await`：几乎所有 I/O 都是异步
- ESM 模块：`import/export`
- 函数式风格更常见（组合函数、纯函数、对象字面量）
- 类型定义和实现分离（接口/类型别名很多）
- 错误处理经常是 `throw` + 上层捕获，而非返回码

## 5) JS/TS 的优点、缺点、注意事项

### 优点

- 开发速度快，生态丰富
- 跨平台部署简单（Node）
- TS 对大型项目非常友好

### 缺点

- 运行时类型不安全（要靠 TS 和测试兜底）
- 依赖生态复杂，版本变化快
- 性能上限通常不如 C++

### 注意事项（高频踩坑）

- `undefined`/`null` 处理必须谨慎
- 异步链路里要注意并发和竞态
- 模块格式（ESM/CJS）不要混乱
- 不要滥用 `any`，否则回到“无类型 JS”
